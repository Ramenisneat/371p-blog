What’s up 👋

The problem from this week was fun to do, like every other week. I relied heavily on iterators, so it was cool to have that concept connect back to what we learnt this week. The problem was cool, because I thought of a neat way of determining the integers, that the largest numbers in the shuffled list will be the integers. The problem was that I was ignoring the multiplicity of the numbers in the list and that was messing me up for a while. I was either using a set which didn’t allow duplicates or using a multiset which I then deleted all instances of an integer out of. Took me a lot of submissions to figure out the multiplicity part. I wish we had more access to the test cases, cause it would’ve made figuring out the problem so much faster. I had to use conditions and return statements placed strategically around my code to figure out what the test cases were testing. I understand that its the nature of competitive programming, but it was frustrating none the less.

The stuff about iterators this week was somewhat confusing. I get the idea of delineating between input, output, random access, etc. But I feel like the real world usage of these was not touched upon enough. I think it makes sense from a security permission, like how rwx perms work on linux. It’s important to expose the least amount of permissions as needed. But I don’t think it was explained super concretely during the lectures, because my mind kept going to “why not just use a random access iterator then”

My pick-of-the-week for this week is kdeconnect. I run arch linux on my laptop (iusearchbtw) and it saddened me before when I saw mac users quickly airdrop files and images to their laptops. I find myself using whatsapp or discord to send files which sucks. kdeconnect works over bluetooth to send and receive files from/to any device. The app’s nifty and you can do a bunch of other stuff as well like running commands, copying clipboards, remote input etc. pretty sick.
